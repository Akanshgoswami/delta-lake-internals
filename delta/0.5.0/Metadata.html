Metadata

Metadata is an action to describe metadata (update) of a delta table (indirectly via Snapshot).

Use DescribeDeltaDetailCommand to review the current metadata of a delta table.

Metadata can be updated for a OptimisticTransactionImpl once (and only when created for the very first time, when readVersion is -1).

Metadata is created when:

  • DeltaLog is requested for the metadata

  • OptimisticTransactionImpl is requested for the snapshotMetadata

  • ConvertToDeltaCommand is requested to performConvert

  • ImplicitMetadataOperation is requested to update a Metadata

Creating Metadata Instance

Metadata takes the following to be created:

  • Table ID (default: a random UUID)

  • Name of the delta table (default: null)

  • Description (default: null)

  • Format

  • Schema (default: null)

  • Names of the partition columns (default: Nil)

  • Configuration (default: empty)

  • Created time (in millis since the epoch)

wrap Method

wrap: SingleAction
wrap is part of the Action contract to wrap the action into a SingleAction.

wrap simply creates a new SingleAction with the metaData field set to this Metadata.
