TransactionalWrite

TransactionalWrite is an abstraction of OptimisticTransactionImpl implementations that can write files out (among other features).

OptimisticTransaction is the default and only known TransactionalWrite in Delta Lake (indirectly as a concrete OptimisticTransactionImpl).

writeFiles turns the hasWritten flag on (true).

writeFiles normalize the given data dataset (based on the partitionColumns of the Metadata).

writeFiles getPartitioningColumns based on the partitionSchema of the Metadata.

writeFiles gets the DelayedCommitProtocol committer for the data path of the DeltaLog.

writeFiles gets the invariants from the schema of the Metadata.

writeFiles requests a new execution ID that is used to track Spark jobs of FileFormatWriter.write (Spark SQL) with a physical query plan of a new DeltaInvariantCheckerExec unary physical operator with the executed plan of the normalized query execution as the child operator.

In the end, writeFiles returns the addedStatuses of the DelayedCommitProtocol committer.

writeFiles is used when:

  • DeleteCommand, MergeIntoCommand, UpdateCommand, and WriteIntoDelta are requested to run

  • DeltaSink is requested to addBatch

hasWritten Flag

hasWritten: Boolean

hasWritten is a flag that is used exclusively to prevent OptimisticTransactionImpl from updating metadata after the transaction has written files out.

hasWritten is turned off (false) by default.

hasWritten is turned on (true) when TransactionalWrite is requested to write files out.
