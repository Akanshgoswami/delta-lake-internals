DeltaLog

DeltaLog is a transactional log of a delta table.

DeltaLog is created (indirectly via DeltaLog.apply utility) when DeltaLog.forTable utility is used.

import org.apache.spark.sql.SparkSession
assert(spark.isInstanceOf[SparkSession])

import org.apache.spark.sql.delta.DeltaLog
val deltaLog = DeltaLog.forTable(spark, dataPath = "/tmp/delta/t1")

scala> println(deltaLog.LAST_CHECKPOINT)
file:/tmp/delta/t1/_delta_log/_last_checkpoint

DeltaLog is a LogStoreProvider.

_delta_log Metadata Directory

DeltaLog uses _delta_log metadata directory under the data path directory (that is specified using DeltaLog.forTable utility).

The _delta_log directory is resolved (in the DeltaLog.apply utility) using the application-wide Hadoop Configuration.

DeltaLog.apply utility uses the given SparkSession to create an Hadoop Configuration instance.

spark.sessionState.newHadoopConf()

Once resolved and turned into a qualified path, the _delta_log directory for the delta table (under the data path directory) is cached for later reuse.

deltaLogCache Cache of _delta_log Fully-Qualified Paths

deltaLogCache: Cache[Path, DeltaLog]

deltaLogCache is a cache of DeltaLogs by their fully-qualified _delta_log metadata directories.

deltaLogCache has a new DeltaLog added in DeltaLog.apply utility.

deltaLogCache is invalidated for a Path in DeltaLog.apply, DeltaLog.invalidateCache utilities and all Paths in DeltaLog.clearCache utility.

Executing Operation (Block) in New Transaction — withNewTransaction Method

withNewTransaction[T](
  thunk: OptimisticTransaction => T): T

withNewTransaction starts a new transaction (that is set active for the whole thread) and executes the given thunk function.

In the end, withNewTransaction makes the transaction no longer active.

withNewTransaction is used when:

  • DeleteCommand, MergeIntoCommand, UpdateCommand, and WriteIntoDelta logical commands are executed

  • DeltaSink is requested to addBatch

  • DeltaTableIdentifier is requested to getDeltaLog
